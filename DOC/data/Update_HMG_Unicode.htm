HMG-UNICODE
Unicode Documentation

 

 

Since version HMG.3.1.0 (2012/11/25), HMG supports both the Unicode and ANSI character sets (for compatibility with previous versions), depending on the choice of compilation in the build of the library. By default HMG supports the Unicode character set (see INCLUDE\SET_COMPILE_HMG_UNICODE.ch).

 

Unicode is the current standard for character set, according to Microsoft documentation:

 

“Unicode is a worldwide character encoding standard that provides a unique number to represent each character used in modern computing, including technical symbols and special characters used in publishing. Unicode is required by modern standards, such as XML and ECMAScript (JavaScript), and is the official mechanism for implementing ISO/IEC 10646 (UCS: Universal Character Set). It is supported by many operating systems, all modern browsers, and many other products. New Windows applications should use Unicode to avoid the inconsistencies of varied code pages and to aid in simplifying localization.”

 

Therefore HMG-Unicode is the future in the xBase programming for Windows. Since version HMG.3.2.0 (2013/12/08), HMG-Unicode is considered stable.

 

To program for HMG-Unicode, set 'Encoding in UTF-8' in your text editor for all the source code files which contain strings in languages using Unicode characters. See the demos in the UNICODE folder in the SAMPLES folder.

 

• General Functions and Commands

 

HMG_SupportUnicode()Returns .T. or .F. Returns true only if HMG is compiled to support the Unicode character set.
 
HMG_CharSetName()Returns "UNICODE" if HMG is compiled for Unicode. Returns "ANSI" if HMG is compiled for ANSI.
 
SET CODEPAGE TO UNICODESets the character code page to UTF-8. If HMG is compiled for Unicode, UTF-8 is the default code page.
 
HMG_IsCurrentCodePageUnicodeReturns TRUE if current code page is UTF-8.
 
IF HMG SUPPORT UNICODE [ RUN | STOP]This is a security command to avoid error in program execution/compilation for programmers that used HMG library with and without support for the Unicode character set.
RUNRun the program only if the HMG library supports Unicode.
STOPStop the execution of the program if the HMG library supports Unicode. Useful when the COMPILE_HMG_UNICODE directive is deactivated when building the HMG library.
 

Note: Programs written entirely for ANSI can be compiled easily with HMG-UNICODE, by adding to the beginning of the function MAIN() the appropriate ANSI code page, e.g. SET CODEPAGE TO SPANISH, without disabling the COMPILE_HMG_UNICODE directive (in file INCLUDE\SET_COMPILE_HMG_UNICODE.ch) and rebuilding the HMG library. Hybrid programs must alternate between the appropriate ANSI code page with UTF-8 code page according to needs.

 

Remember: To develop applications that support Unicode, you should replace ALL functions that ONLY support the ANSI character set with equivalent Unicode functions.
