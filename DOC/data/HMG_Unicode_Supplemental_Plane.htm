Unicode Supplemental Plane Characters in HMG
Handling Unicode Supplemental Plane Characters in HMG

 

 

 

Most Unicode text uses only code points in the lowest plane, the BMP (Basic Multilingual Plane). Code points in the BMP range from 0 to 0xFFFF.

 

Other code points, 0x10000 and higher, are in one of the supplemental planes. The first supplemental plane, the SMP (Supplemental Multilingual Plane), is most frequently used.

 

Harbour functions for handling UTF-8 support only the BMP directly. To support supplemental plane characters, Harbour uses a variant of UTF-8 called CESU-8. UTF-8 and CESU-8 are identical for BMP characters, but they use completely different mechanisms for encoding supplemental plane characters.

 

Since Windows uses UTF-16, and HMG uses Harbour to translate between UTF-16 and CESU-8, you must translate between CESU-8 and UTF-8 if you use supplemental plane characters in your applications.

 

• When you assign a string containing a supplemental plane character into a Windows object, you must translate the string from UTF-8 to CESU-8. See HMG_CESU8().
 
• When you retrieve a string containing a supplemental plane character from a Windows object, you may want to translate the string from CESU-8 to UTF-8. You must translate the string if you want to use it outside your application, since there is very little software, other than Harbour, that directly supports CESU-8. See HMG_UNCESU8().

 

Examples:

 

Win_1.Editbox_1.VALUE := HMG_CESU8( MEMOREAD( 'MyFile.txt' ) )
 
HB_MEMOWRIT( 'MyFile.txt', HMG_UNCESU8( Win_1.Editbox_1.VALUE ) )

 

To work around some of the limitations of Harbour UTF-8 functions, HMG has several analogs of Harbour functions. These HMG functions correctly handle all UTF-8 characters.
